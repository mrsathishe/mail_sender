MailRelay — Project NotesDraft · Work in progressWhat is itA middleware service that lets applications send emails through their own mail account (e.g. Gmail).Apps call a simple REST API instead of setting up mail infrastructure themselves.Useful for contact forms, support forms, and any app that needs to send email.
How it worksApp sends a POST request to the MailRelay API with the email payload.Request is authenticated using a secret key unique to each registered app.MailRelay validates the key, then forwards the email via the app's linked mail provider.
AuthenticationEach app gets a unique secret key on registration.Secret key is passed in the request header to authenticate every API call.Key generation and management to be decided later.
App onboardingRegister app on the dashboard — provide app name and basic details.Connect a mail provider (e.g. Gmail via OAuth, or SMTP credentials).Secret key is generated and shown once — store it securely in environment variables.Full onboarding flow details to be defined later.
Tech stackNext.js — framework for both the dashboard and the REST API routes.React — UI for the dashboard and onboarding flow.MongoDB — stores app registrations, secret keys, and send logs.Zustand — client-side state management for the dashboard.
To decide laterSecret key generation strategy and rotation policy.Full app onboarding flow and UX.Supported mail providers beyond Gmail.Rate limiting and usage quotas per app.
